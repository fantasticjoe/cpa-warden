# Security Policy

## Supported Versions

The project is still early-stage. The latest `0.1.x` release line is treated as supported for security fixes.

## Reporting A Vulnerability

Do not open a public GitHub issue for suspected security problems.

When reporting a vulnerability:

- do not include real `token` values
- do not include real account identifiers
- do not attach raw auth files, SQLite databases, or exported account JSON from live environments
- do not paste logs that contain operational secrets or sensitive identifiers

Preferred reporting path:

1. Use GitHub's private vulnerability reporting for this repository if it is available.
2. If private reporting is not available, contact the repository owner privately before any public disclosure.

Please include:

- affected version or commit
- clear reproduction steps
- expected impact
- sanitized logs or screenshots if they help explain the issue

Security reports will be reviewed privately first. Public disclosure should wait until the issue has been confirmed and a fix or mitigation is available.
